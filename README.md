# Fire Compliance Tracker

Система отслеживания пожарного комплаенса арендованных офисов в мультистрановом портфеле.
Один экран — все объекты с автоматическим расчётом состояния по четырём измерениям:
сертификаты · системы ПБ · обучение · тренировки эвакуации.

## Возможности

- **Реестр объектов** с группами риска A–D (по привязке сертификата), приоритетами P1–P3 и статусом достоверности данных
- **Системы ПБ**: каталог из 10 позиций (огнетушители, АУПС, спринклеры, кухонные системы F/K, Li-ion, Hassantuk и др.) с датами проверок; следующий срок считается автоматически по интервалу
- **Нормы по странам**: интервалы инструктажей, ПТМ, тренировок берутся из локальных требований (СНГ, ОАЭ, Китай, Боливия…), для остальных — NFPA 10/25/72/101 как внутренний стандарт
- **Обучение и эвакуация**: приказ об ответственном, инструктажи, ПТМ (где требуется), журнал тренировок
- **Коворкинги**: если сертификат держит оператор, трекер требует не собственный сертификат, а копию от оператора
- **Хранение**: единый JSON-документ с номером схемы и миграциями, журнал изменений, экспорт/импорт для бэкапов и передачи

## Запуск

```bash
npm install
npm run dev        # http://localhost:5173
npm run build      # сборка в dist/
```

## Деплой на GitHub Pages

В репозитории уже лежит workflow `.github/workflows/deploy.yml`:
1. Settings → Pages → Source: **GitHub Actions**
2. Пуш в `main` — сборка и публикация происходят автоматически

## Данные

- Стартовые объекты: `src/data/seed.json`. **Перед публичной публикацией замените или очистите этот файл** (`[]`) — либо держите репозиторий приватным: файл содержит внутренние данные об объектах.
- Рабочие данные хранятся в `localStorage` браузера (ключ `fintrent-fire-doc-v3`) и не попадают в git.
- Кнопка **Экспорт** выгружает полный документ `{schema, objects, log}` — используйте как бэкап; **Импорт** принимает и полный документ, и простой массив объектов.
- Формат объекта и логика статусов описаны в `src/App.jsx` (константы `SYSTEMS_CATALOG`, `NORMS`, функция `assess`).

## Ограничения

Однопользовательский инструмент: данные живут в браузере, одновременное редактирование не поддерживается (при импорте побеждает последняя версия целиком). Для командной работы потребуется бэкенд — модель данных к этому готова.

## Дисклеймер

Инструмент помогает отслеживать статусы, но не заменяет локальную юридическую проверку. Номера актов и интервалы для стран со статусом 🟡/🔴 подтверждайте по первоисточникам.
